Dua tahun gue ngoding hampir semuanya pakai Laravel. Terus gue nyoba Next.js, dan pandangan gue soal web development berubah lumayan banyak.

## Background

Gue mulai serius coding sekitar 2023. Laravel jadi framework pertama yang gue pelajari secara mendalam — routing, Eloquent ORM, Blade templating, semua gue kuasai pelan-pelan.

Masalahnya? Setiap kali lihat portofolio developer lain, mereka pada pakai React. Gue merasa ketinggalan.

## First Impression Next.js

Jujur, awalnya **bingung banget**. Mental model Laravel itu server-centric — request masuk, proses di server, kirim HTML. Next.js? Lo harus mikirin mana yang jalan di server, mana di client.

```tsx
// Ini jalan di server (default di App Router)
async function BlogPage() {
  const posts = await getPosts(); // fetch langsung, no useEffect
  return <div>{posts.map(p => <PostCard key={p.slug} post={p} />)}</div>;
}
```

Konsep `"use client"` vs Server Component itu butuh waktu untuk benar-benar ngerti.

## Yang Lebih Enak di Next.js

- **DX (Developer Experience)** — Hot reload-nya kenceng banget
- **File-based routing** — Bikin route baru tinggal bikin file
- **Built-in optimization** — Image optimization, font loading, semua udah diurus
- **Full-stack dalam satu codebase** — API routes + frontend bareng

## Yang Masih Lebih Enak di Laravel

- **Database operations** — Eloquent masih lebih intuitif daripada Prisma/Drizzle
- **Authentication** — Laravel Sanctum/Breeze udah lengkap banget
- **Queue, jobs, events** — Laravel punya sistem ini built-in yang mature
- **PPDB/Sistem Informasi** — Untuk project CRUD berat, Laravel tetap jadi pilihan gue

## Kesimpulan

Sekarang gue pakai **keduanya sesuai konteks**:
- **Next.js** → portofolio, landing page, app yang butuh performa frontend tinggi
- **Laravel** → sistem informasi, backend API, project CRUD kompleks

Nggak ada yang lebih baik. Yang ada adalah *tools yang tepat untuk masalah yang tepat*.
